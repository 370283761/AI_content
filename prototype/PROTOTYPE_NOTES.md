# PROTOTYPE — 用完即删

## 要回答的问题

1. 用户能否理解并完成“小说 -> 设定 -> 剧集 -> 剧本 -> 分镜 -> 即梦生产 -> 交付物”的流程？
2. 即梦生产阶段更适合向导、专业工作台还是任务看板？

## 三个生产页方案

- `?variant=A`：步骤向导，强调单镜头连续操作。
- `?variant=B`：专业工作台，强调上下文和编辑效率。
- `?variant=C`：任务看板，强调批量生产进度。

生产页右下角提供布局切换；输入框未聚焦时也可使用左右方向键切换。

## 启动

```bash
npm run prototype
```

打开 `http://localhost:4173`。

## 验证后记录

- 最终选择：正式生产页以 B（专业工作台）为主。
- 保留的设计：前期创作保留 A 的步骤向导；生产页吸收 C 的状态筛选和进度统计。
- 需要重做的交互：正式工程需使用动漫短剧示例数据，并连接持久化、任务状态和真实提示词版本。

本目录没有生产级数据存储、鉴权、AI 调用、错误处理或测试，不应直接升级为正式应用。
