## 模块职责

### 业务模块（`src/modules/`）

| 模块 | 职责 | 关键文件 |
|---|---|---|
| **projects** | 默认租户下的项目 CRUD、原始输入、继续制作 | `service.ts`, `schemas.ts` |
| **content** | 圣经、角色、关系、地点、分集、场景、镜头人工 CRUD | `service.ts`, `schemas.ts` |
| **ingestion** | 小说生成、故事分析、原文保存 | `novelist.service.ts`, `analyst.service.ts` |
| **bible** | 故事/角色/视觉圣经、锁定、关系 | `character.service.ts`, `style.service.ts` |
| **planning** | 整季与分集规划、一致性检查 | `planner.service.ts` |
| **script** | 单集剧本、场景 CRUD、局部 AI 编辑 | `writer.service.ts`, `local-edit.service.ts` |
| **shot** | 分镜拆解、镜头编辑、连续性检查 | `storyboard.service.ts`, `continuity.service.ts` |
| **prompt** | 提示词生成、版本管理、Seedance 适配 | `prompt-smith.service.ts`, `optimizer.service.ts` |
| **production** | 生产状态机、结果上传、失败反馈 | `production.service.ts` |
| **versioning** | 影响传播、锁定守卫、版本恢复 | `impact.service.ts`, `lock-guard.ts` |
| **export** | 导出生产包（XLSX/TXT/MD/ZIP） | `export.service.ts` |

### AI 相关（`src/ai/`）

| 目录 | 职责 |
|---|---|
| **gateway** | 统一模型网关、Schema 校验、重试、成本记录 |
| **providers** | Claude 与 GPT-5.6 SDK 封装 |
| **adapters** | Seedance 2.0 提示词适配器 |
| **schemas** | 所有 AI 输出的 Zod Schema 定义 |
| **prompts** | AI 提示词模板（Markdown 格式） |

### 共享基础设施（`src/shared/`）

| 目录 | 职责 |
|---|---|
| **db** | Prisma Client 单例 |
| **queue** | BullMQ 队列与 Worker |
| **storage** | MinIO/S3 对象存储封装 |
| **logger** | 结构化日志（pino） |
| **config** | 环境变量加载与校验 |

### 前端（`app/` + `src/components/`）

| 目录 | 职责 |
|---|---|
| **app/** | Next.js 页面路由与 API Route Handlers |
| **src/components** | 可复用 React 组件与阶段编辑器 |
| **src/hooks** | 自动保存等自定义 Hooks |

## 命名约定

- **文件名**：kebab-case（`project-service.ts`）
- **类型/接口**：PascalCase（`ProjectService`, `IProjectRepository`）
- **函数/变量**：camelCase（`createProject`, `projectList`）
- **常量**：SCREAMING_SNAKE_CASE（`MAX_PROJECT_SIZE`）
- **数据表**：snake_case（Prisma 中使用 `@@map` 显式配置）

```text
  ↓

  ↓

  ↓
src/ai/providers (Claude/GPT-5.6)
  ↓
src/shared/* (Prisma/Queue/Storage/Logger)
```

---

## 模块间通信规则

1. **API 层不直接调用 AI 网关**，必须通过业务服务。
2. **业务服务不直接调用 Prisma Client**，应通过 Repository 或 Service 内部封装。
3. **AI 网关不查询数据库**，所有上下文由业务模块组装传入。
4. **平台适配器只出现在 `src/ai/adapters/`**，业务代码禁止引用。
5. **锁定检查必须在写入前调用 `lockGuard`**（`src/modules/versioning/lock-guard.ts`）。

---

## 新增模块/文件时的注意事项

1. **新增业务模块**：在 `src/modules/` 下创建目录，并更新本文档。
2. **新增 AI Agent**：
   - 在 `src/ai/schemas/` 添加 Schema
   - 在 `src/ai/prompts/` 添加模板
   - 在 [AGENTS.md](./AGENTS.md) 添加规范
3. **新增 API 端点**：
   - 在 `app/api/` 对应路径创建 `route.ts`
   - 更新 [TECHNICAL_DESIGN.md 第 9.2 章](./docs/TECHNICAL_DESIGN.md#92-核心接口清单)
4. **新增数据表**：
   - 修改 `prisma/schema.prisma`
   - 生成迁移：`pnpm prisma migrate dev --name <描述>`
   - 更新 [TECHNICAL_DESIGN.md 第 5 章](./docs/TECHNICAL_DESIGN.md#5-数据模型设计)
