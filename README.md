# AI 视频剧集创作平台

> 帮助创作者把故事创意或平台生成的小说章节转化为可在即梦（Seedance 2.0）逐镜头生产的视频剧集方案，并管理剧本、分镜、提示词、一致性和制作进度的 AI 创作工作台。

**当前状态**：需求与架构设计阶段（M0）  
**版本**：V0.2 需求基线  
**更新时间**：2026-07-16

## 产品定位

**目标用户**：动漫短剧创作者（MVP 首发优先）  
**核心价值**：AI 辅助 + 人机协作，从故事创意到可执行的视频生产包  
**交付物**：单集生产包（剧本 MD + 分镜 XLSX + 即梦提示词 MD/TXT + 制作进度 + ZIP 打包）

**MVP 核心链路**：

```text
故事创意（或平台生成的小说章节）
  → 项目设定
  → 故事/角色/视觉圣经
  → 整季与分集规划
  → 单集剧本
  → 分镜镜头（8-15 个）
  → 即梦（Seedance 2.0）提示词
  → 复制到即梦生成
  → 结果回填与提示词迭代
  → 单集生产包导出（XLSX/TXT/MD/ZIP）
```

## 核心功能（MVP）

### 1. 项目管理
- 创建项目（故事创意 / 小说生成 / 小说导入三种入口）
- 项目列表与继续制作
- 自动保存（1s 节流）
- 单项目容量 1GB 限制

### 2. 内容生成与分析
- 小说章节生成（NovelistAgent）
- 故事结构分析（AnalystAgent）
- 原始输入保留，不被 AI 覆盖

### 3. 故事圣经
- 故事圣经（主题、世界观、核心冲突）
- 角色圣经（外貌、性格、关系、标志物、锁定字段）
- 视觉圣经（画面风格、色彩、光线、比例）

### 4. 剧集规划
- 整季故事弧与分集卡片生成（PlannerAgent）
- 剧情一致性检查
- 单集重生成与锁定保护

### 5. 单集剧本
- 结构化场景生成（WriterAgent）
- 局部 AI 编辑（优化对白、加强冲突、缩短时长等）
- 时长估算与建议镜头数

### 6. 分镜与镜头
- 自动拆分镜头（StoryboardAgent，8-15 个）
- 镜头字段编辑（景别、角度、运镜、动作、表情、光线等）
- 连续性检查（ContinuityAgent）
- 生成风险提示

### 7. 提示词生成
- 即梦（Seedance 2.0）视频提示词生成（PromptSmithAgent）
- 首帧提示词生成
- 提示词版本管理
- 一键复制

### 8. 即梦生产模式
- 镜头生产状态机（待生成/已提交/已采用/待修改/已废弃）
- 生成结果上传
- 失败反馈 → 提示词自动优化（OptimizerAgent）
- 制作进度汇总

### 9. 影响传播
- 上游修改自动检测引用关系
- 影响范围提示（锁定项标记）
- 用户确认后同步未锁定内容

### 10. 导出
- 单集生产包（XLSX/TXT/MD 格式）
- 导出前检查（缺失项警告）
- ZIP 打包下载

## 已确认的产品决策（第 20 章）

1. 视频平台：**即梦 AI**（Seedance 2.0）
2. 优先支持：**动漫短剧**
3. 内容输入：**小说生成 + 小说导入**（不支持文件上传）
4. 单集配置：**推荐模板 + 用户可配置**
5. 图片生成：**暂不支持**（仅生成提示词）
6. 导出格式：**XLSX/TXT/MD**（不含 PDF/DOCX）
7. AI 供应商：**Claude + GPT-5.6**（可配置切换）
8. 账号系统：**无需账号**（本地/单租户模式）
9. 容量上限：**单项目 1GB**
10. 内容安全：**暂不作为首发重点**

## 快速启动（本地开发）

详细步骤见 [本地开发环境.md](./本地开发环境.md)。

```bash
# 1. 安装依赖
pnpm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入 Claude 和 GPT-5.6 的 API Key

# 3. 启动依赖服务（Postgres、Redis、MinIO）
docker compose up -d

# 4. 数据库迁移
pnpm prisma generate
pnpm prisma migrate dev

# 5. 启动开发服务器
pnpm dev
```

访问 `http://localhost:3000`。

## 开发流程

本项目采用 **7 步开发流程**（详见 [CLAUDE.md](./CLAUDE.md)）：

1. **需求确认**：查阅历史 commit、功能清单、产品需求，明确边界
2. **需求拆解**：拆分为可独立验证的最小功能点
3. **需求实现**：先读后写、遵守技术方案、逐文件交付
4. **需求归档**：Git commit + 同步文档（功能清单、README 等）
5. **需求自测**：单元测试 + 手工测试 + 测试用例归档
6. **提交变更**：自查清单 + 推送代码
7. **回归检查**：影响范围分析 + 自动化/手工回归测试
